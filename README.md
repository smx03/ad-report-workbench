# 信息流投放工作台

纯前端的素材复盘、广告日报、时报和任务维护工具，可通过 GitHub Pages 访问。

## 素材周度复盘

1. 上传一份包含内广、外广素材表现和任务映射的抖音精选周表。
2. 按需填写内外广目标成本、最低有效样本和下周总产能。
3. 浏览器本地完成重复素材聚合、表现分层、内容方向归纳和下周创意 Brief。
4. 复制 Brief 给素材团队，或导出完整素材清单继续复核。

当前版本依据素材标题、标签、任务类型和投放数据归纳内容方向；视频画面、前三秒口播和字幕节奏需在后续接入视频文件后分析。

## 日报流程

1. 上传日报日、前一日、七日前三份全部账户 Excel。
2. 上传本次使用的最新账户匹配表。
3. 浏览器本地解析、校验、计算并生成拉新/卸载两张 PNG。

四份 Excel 不会发送给 GitHub 或任何后端。原始文件仅存在于当前页面内存，刷新或关闭页面后释放。

## 时报流程

上传一份包含今天、昨天、前天分时数据的 Excel，选择发送时刻后生成两张时报图片。该流程同样只在浏览器本地运行。

## 一站式日期维护

工作台使用纯前端页面生成更新预览，`extension/` 中的 Chrome 扩展负责在已登录的一站式页面内读取和修改日期。扩展不读取、保存或上传账号密码。

### 本地安装

1. 运行 `pnpm preview`，打开 `http://127.0.0.1:4319/`。
2. Chrome 打开 `chrome://extensions/`，开启右上角“开发者模式”。
3. 点击“加载已解压的扩展程序”，选择本项目的 `extension/` 目录。
4. 刷新工作台，进入左侧“一站式日期维护”。
5. 手动登录一站式，先扫描任务、生成预览，再勾选执行。

扩展代码更新后，需在 `chrome://extensions/` 中点击该扩展的“重新加载”，再刷新工作台和已打开的一站式页面。
工作台会显示扩展版本、后台协议和一站式页面脚本状态；任意一层版本不一致时会禁止预览和执行。

首次试运行只选择一条可核对的任务。确认一站式成功提交且无其他字段变化后，再扩大到全部任务。

## 本地预览

```bash
pnpm preview
```

打开 `http://127.0.0.1:4319/`。开发时使用 HTTP 预览，不要直接双击 `index.html`。

## 发布

`.github/workflows/pages.yml` 会在 `main` 分支更新后将 `public/` 发布到 GitHub Pages。

## 安全边界

- 仓库不包含账户匹配表、原始数据、SQLite、历史日报或上传文件。
- 页面只托管 HTML、CSS、JavaScript，以及浏览器版 ExcelJS 和 JSZip。
- 纯前端规则会下载到访问者浏览器，因此计算参数不属于服务器密钥。
- 一站式扩展仅对 `usergrowth.com.cn/onestop/` 生效；预览不会提交，执行前必须在工作台二次确认。
